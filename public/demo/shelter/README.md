# AI 監測範例照片

AI Camera 避難所監測節點的 simulation 引擎會「輪播」這個資料夾的照片，
真的呼叫 `/api/vision`（mode=shelter）做影像辨識，產生避難所人數與物資狀態。

請放入以下檔名的避難所室內照片（任意真實照片即可，AI 會依畫面判斷）：

| 檔名 | 建議畫面 | demo 對應避難所 |
|---|---|---|
| `normal-1.jpg` | 一般避難所大廳、人潮寬鬆、物資正常 | 其餘一般避難所 |
| `supplies.jpg` | 毛毯 / 民生物資堆放區 | 其餘一般避難所 |
| `crowded.jpg` | 人潮擁擠、接近額滿 | TN-E-009 東光國小（接近額滿）|
| `food-low.jpg` | 糧食 / 食物箱所剩不多 | TN-E-004 成大工學院（糧食不足）|
| `water-low.jpg` | 飲水區瓶裝水所剩不多 | TN-E-001 東區區公所（飲水偏低）|
| `dark-power-out.jpg` | 室內昏暗 / 停電 | TN-E-008 市立醫院（電力不穩）|

檔名清單定義在 `src/utils/shelterAi.ts` 的 `SAMPLE_PHOTOS`；
避難所 ↔ 照片 ↔ 異常劇本對應定義在同檔的 `DEMO_PROFILES`。

> demo 設計：模擬模式下固定這 4 所呈現不同異常（2 紅 2 黃），其餘維持正常；
> 黃色（偏低/不穩）也算「警戒」異常，會送指揮中心，只是以黃色顯示、不是紅色。

> 沒有放照片也沒關係：抓不到照片 / 未設定 `HF_TOKEN` / 離線時，
> 監測引擎會自動 fallback 成「演算法模擬讀數」，demo 仍可正常運作，
> 只是不會真的跑影像辨識。
